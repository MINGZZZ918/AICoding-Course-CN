# 第一章：起步 - 认识 AI 编程

## 1.1 什么是 AI 编程

### 教学目标
- 理解 AI 编程不是「学编程」，而是「学会指挥 AI」
- 消除对编程的恐惧感
- 建立正确的期待

### 核心概念

**传统编程 vs AI 编程**

| 传统编程 | AI 编程 |
|----------|---------|
| 你需要学习编程语言 | 你只需要会说人话 |
| 你需要记住语法 | AI 帮你写语法 |
| 一个错误可能卡你几小时 | 把错误告诉 AI，它帮你修 |
| 门槛高，学习曲线陡 | 门槛低，边做边学 |

**AI 编程的工作流程**
```
你的想法 → 用文字描述 → AI 生成代码 → 你看效果 → 不满意就继续说 → 直到满意
```

**你的角色**
- 不是程序员，是「产品经理」
- 你负责：想清楚要什么、描述清楚、检查结果
- AI 负责：写代码、改代码、解释问题

### 案例展示
> 演示：用 5 分钟让 AI 生成一个「个人介绍网页」
>
> 展示从描述需求到看到成品的完整过程，让学员直观感受 AI 编程的威力

### 本节要点
1. AI 编程 = 描述想法 + 审核结果 + 调整优化
2. 你不需要懂代码，但需要懂得如何描述你的需求
3. AI 会犯错，但你可以让它改，这很正常

## 1.2 安装你的 AI 助手 - Cursor

### 教学目标
- 成功安装 Cursor
- 完成账号注册
- 认识基本界面

### 什么是 Cursor

Cursor 是一个「内置了 AI 的代码编辑器」。

你可以把它想象成一个「写代码的 Word」，但是里面住着一个 AI 助手，你随时可以跟它对话，让它帮你写代码。

### 安装步骤

**Step 1：下载**
- 访问 https://cursor.com
- 点击 Download，选择你的系统（Mac / Windows）
- 下载完成后，双击安装

**Step 2：安装（Mac）**
- 打开下载的 .dmg 文件
- 把 Cursor 拖到 Applications 文件夹
- 从启动台打开 Cursor

**Step 3：安装（Windows）**
- 运行下载的 .exe 安装包
- 按提示完成安装
- 从开始菜单打开 Cursor

### 账号注册与套餐

**注册账号**
- 首次打开会提示登录
- 可以用 Google 账号或邮箱注册
- 建议使用常用邮箱，方便找回

**套餐选择建议**

| 套餐 | 价格 | 适合人群 |
|------|------|----------|
| Free | 免费 | 先体验，每月有限额 |
| Pro | $20/月 | 认真学习和使用，推荐 |

> 建议：先用免费版体验，确定要认真学再升级

### 界面认识

打开 Cursor 后，你会看到：

```
┌─────────────────────────────────────────────────────────┐
│  菜单栏                                                  │
├──────────┬──────────────────────────────┬───────────────┤
│          │                              │               │
│  文件    │      代码编辑区               │    AI 对话    │
│  列表    │      （写代码的地方）          │    面板       │
│          │                              │               │
│          │                              │               │
├──────────┴──────────────────────────────┴───────────────┤
│  底部状态栏                                              │
└─────────────────────────────────────────────────────────┘
```

**你需要知道的几个区域**
1. **文件列表**（左侧）：显示你项目里的文件
2. **代码编辑区**（中间）：代码会显示在这里
3. **AI 对话面板**（右侧）：和 AI 聊天的地方，按 `Cmd+L`（Mac）或 `Ctrl+L`（Windows）打开

### 动手练习
1. 成功打开 Cursor

## 1.3 第一次对话：让 AI 帮你写代码

### 教学目标
- 学会和 AI 对话
- 理解 AI 回复的内容
- 完成第一个 AI 生成的网页

### 打开对话面板

- Mac：按 `Cmd + L`
- Windows：按 `Ctrl + L`
- 或者点击右侧的聊天图标

### 用中文描述你想要的

**示例对话 1：打招呼**
```
你：你好，你能帮我做什么？

AI：你好！我可以帮你：
- 生成代码
- 解释代码
- 修改代码
- 回答编程问题
...
```

**示例对话 2：生成网页**
```
你：帮我创建一个简单的个人介绍网页，包含：
- 我的名字：小明
- 一句自我介绍
- 我的爱好列表

AI：好的，我来帮你创建...（然后会给你代码）
```

### 理解 AI 的回复

AI 回复通常包含：

1. **代码块**：灰色背景的部分，这就是代码
2. **解释说明**：告诉你这段代码是什么
3. **操作按钮**：Apply（应用代码）、Copy（复制）

### 把代码变成真实网页

**方法一：使用 Apply 按钮**
1. 先创建一个文件夹作为项目
2. 用 Cursor 打开这个文件夹
3. 在对话中让 AI 生成代码
4. 点击代码块右上角的「Apply」
5. 代码会自动应用到文件

**方法二：使用 Composer**
1. 按 `Cmd+I`（Mac）或 `Ctrl+I`（Windows）打开 Composer
2. 描述你要什么
3. AI 会直接帮你创建文件

### 动手练习：生成一个计算器网页

**Step 1：创建项目文件夹**
- 在桌面创建一个文件夹，命名为「我的第一个项目」
- 用 Cursor 打开这个文件夹（File → Open Folder）

**Step 2：和 AI 对话**
打开 Composer（`Cmd+I`），输入：

```
帮我创建一个简单的网页计算器，要求：
1. 能进行加减乘除
2. 有数字按钮 0-9
3. 界面好看一点
4. 创建一个 index.html 文件
```

**Step 3：查看结果**
- AI 会生成代码并创建文件
- 在文件列表中找到 index.html
- 右键点击 → 选择「在浏览器中打开」
- 看看你的计算器！

**Step 4：修改优化**
不满意？继续对话：
```
把按钮改成圆形的，颜色用蓝色
```

### 本节要点
1. `Cmd+L` 打开对话，`Cmd+I` 打开 Composer
2. 用中文描述你的需求，越具体越好
3. 看到代码后点 Apply 或让 AI 直接创建文件
4. 不满意就继续说，AI 会帮你改

# 第二章：核心技能 - 和 AI 高效协作

## 2.1 学会「描述」你的想法

### 教学目标
- 理解什么是好的需求描述
- 掌握描述的基本结构
- 通过对比学会优化描述

### 为什么描述很重要

AI 只能根据你说的话来理解你要什么。

- 说得模糊 → AI 瞎猜 → 结果不满意
- 说得清楚 → AI 明白 → 一次到位

### 好描述 vs 差描述

**例子 1：做一个按钮**

| 差的描述 | 好的描述 |
|----------|----------|
| 帮我做个按钮 | 帮我做一个按钮，要求：<br>- 蓝色背景，白色文字<br>- 写着「提交」<br>- 鼠标放上去会变深一点<br>- 点击后显示「提交成功」 |

**例子 2：做一个页面**

| 差的描述 | 好的描述 |
|----------|----------|
| 做个好看的首页 | 帮我做一个个人博客首页：<br>- 顶部有导航栏，包含：首页、文章、关于我<br>- 中间是文章列表，每篇显示标题和日期<br>- 底部有版权信息<br>- 风格简洁现代，主色用深蓝色 |

### 描述的基本结构

一个好的描述包含三部分：

```
1. 我要做什么（目标）
   → 帮我做一个 XXX

2. 具体要求（细节）
   → 需要有 A、B、C 功能
   → 样式是什么样的

3. 参考说明（可选）
   → 类似于 XXX 那种
   → 风格参考 XXX
```

**模板**
```
帮我创建一个 [东西名称]，要求：
1. [功能要求 1]
2. [功能要求 2]
3. [功能要求 3]
- 风格：[风格描述]
- 颜色：[颜色偏好]
```

### 动手练习

用三种不同方式描述「一个待办事项列表」，对比 AI 生成的结果：

**版本 1（模糊）**
```
做个待办清单
```

**版本 2（一般）**
```
做一个待办事项列表，可以添加和删除任务
```

**版本 3（清晰）**
```
帮我做一个待办事项列表网页：

功能：
1. 可以输入新任务，按回车添加
2. 每个任务前面有勾选框，打勾表示完成
3. 完成的任务显示删除线
4. 有删除按钮可以删除任务
5. 任务数据保存在本地，刷新不丢失

样式：
- 简洁干净的设计
- 主色调用绿色
- 卡片式布局，有阴影效果
```

观察三个版本生成的结果差异。

### 本节要点
1. 描述越具体，AI 理解越准确
2. 使用「目标 + 细节 + 参考」的结构
3. 不怕啰嗦，宁可多说，不要少说

## 2.2 Cursor 三大功能

### 教学目标
- 掌握 Chat、Composer、Inline Edit 三种模式
- 知道什么情况用哪种

### 功能一：Chat（聊天）

**快捷键**：`Cmd+L`（Mac）/ `Ctrl+L`（Windows）

**用途**：
- 问问题
- 让 AI 解释代码
- 让 AI 生成代码片段
- 讨论方案

**适合场景**：
- 「这段代码是什么意思？」
- 「怎么实现 XXX 功能？」
- 「帮我写一个 XXX 函数」

**使用示例**
```
你：这个错误是什么意思？[粘贴错误信息]

AI：这个错误表示...解决方法是...
```

### 功能二：Composer（作曲家）

**快捷键**：`Cmd+I`（Mac）/ `Ctrl+I`（Windows）

**用途**：
- 创建新文件
- 同时修改多个文件
- 做比较大的改动

**适合场景**：
- 「帮我创建一个完整的登录页面」
- 「把这个项目的所有按钮样式统一改掉」
- 「创建一个新的 XXX 组件」

**使用示例**
```
你：创建一个用户注册页面，包含：
- 用户名输入框
- 密码输入框（两次确认）
- 注册按钮
- 表单验证
放在 register.html 文件里

AI：[直接创建文件并写好代码]
```

### 功能三：Inline Edit（行内编辑）

**快捷键**：选中代码后按 `Cmd+K`（Mac）/ `Ctrl+K`（Windows）

**用途**：
- 修改选中的代码
- 局部优化
- 小范围重构

**适合场景**：
- 「把这个函数改成 XXX」
- 「优化这段代码」
- 「把这个颜色改成红色」

**使用示例**
1. 选中一段代码
2. 按 `Cmd+K`
3. 输入：「把背景色改成渐变色，从蓝到紫」
4. AI 直接修改选中的部分

### 三种功能对比

| 功能 | 快捷键 | 适合场景 | 特点 |
|------|--------|----------|------|
| Chat | `Cmd+L` | 问问题、讨论、小代码 | 对话式，灵活 |
| Composer | `Cmd+I` | 创建文件、大改动 | 直接操作文件 |
| Inline Edit | `Cmd+K` | 局部修改 | 精准、快速 |

### 简单判断标准

```
要创建新文件？ → Composer
要改一小部分代码？ → Inline Edit
要问问题或讨论？ → Chat
不确定？ → Chat 先聊，需要改文件时用 Composer
```

### 动手练习

1. 用 **Composer** 创建一个 `index.html` 文件，内容是一个简单的自我介绍
2. 用 **Chat** 问 AI：「怎么让文字居中显示？」
3. 选中某段代码，用 **Inline Edit** 把颜色改成你喜欢的颜色

## 2.3 当 AI 出错时怎么办

### 教学目标
- 接受「AI 会出错」这个事实
- 掌握让 AI 修复错误的方法
- 学会看懂基本的错误信息

### AI 会犯错，这很正常

就像人会犯错一样，AI 也会：
- 理解错你的意思
- 写出有 bug 的代码
- 给出过时的方案

**重要心态**：出错不可怕，让它改就行。

### 三种应对方式

**方式 1：补充说明，重新生成**

当 AI 理解错了你的意思：

```
你：做一个日历
AI：[生成了一个显示当月日期的日历]

你：不对，我要的是可以记事的日历，能在每天上面添加事件

AI：明白了，我来重新做...[生成带事件功能的日历]
```

**方式 2：指出具体问题，让 AI 修复**

当代码有 bug 或不符合要求：

```
你：这个按钮点击没反应，帮我检查一下

AI：我看一下代码...问题在于...我来修复
```

```
你：颜色不对，我要的是深蓝色，不是浅蓝色

AI：抱歉，我来修改颜色...
```

**方式 3：把错误信息发给 AI**

当页面报错或功能不正常：

```
你：运行后浏览器显示这个错误：
[粘贴错误信息]
帮我修复

AI：这个错误是因为...我来修复这个问题
```

### 常见错误类型

**1. 页面空白**
- 可能原因：代码有语法错误
- 解决方法：把现象告诉 AI，让它检查

**2. 样式不对**
- 可能原因：CSS 没写对或没生效
- 解决方法：描述你期望的样式，让 AI 修改

**3. 功能不工作**
- 可能原因：JavaScript 代码有问题
- 解决方法：打开浏览器控制台（F12），把红色错误信息发给 AI

**4. 和预期不一样**
- 可能原因：你的描述 AI 理解偏了
- 解决方法：更详细地说明你要什么

### 查看错误信息的方法

**浏览器控制台**
1. 打开你的网页
2. 按 `F12` 或右键选「检查」
3. 点击「Console」（控制台）标签
4. 红色的文字就是错误信息
5. 复制发给 AI

### 动手练习

**故意制造一个错误，学会修复流程**

1. 让 AI 生成一个带按钮的页面
2. 手动把代码里某个单词删掉几个字母（制造错误）
3. 刷新页面，看看会发生什么
4. 打开控制台，复制错误信息
5. 把错误信息发给 AI，让它修复

### 本节要点
1. AI 出错是正常的，关键是会让它改
2. 描述越具体，AI 修复越准确
3. 善用浏览器控制台获取错误信息
4. 不要怕出错，每次修复都是学习

## 3.2 环境准备

### 教学目标
- 注册微信小程序账号
- 安装微信开发者工具
- 创建项目并用 Cursor 打开

### Step 1：注册小程序账号

**流程**
1. 访问 https://mp.weixin.qq.com
2. 点击「立即注册」
3. 选择「小程序」
4. 选择「个人」主体类型
5. 填写信息，完成注册
6. 登录后，在「开发」→「开发设置」中找到 **AppID**，记下来

> 注意：一个身份证可以注册 5 个小程序

### Step 2：下载微信开发者工具

1. 访问 https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 下载对应系统版本
3. 安装并打开

### Step 3：创建小程序项目

1. 打开微信开发者工具
2. 点击「+」创建项目
3. 填写信息：
   - 项目名称：随便起（如「我的纪念日」）
   - 目录：选择一个文件夹存放项目
   - AppID：填入刚才记录的 AppID
   - 开发模式：小程序
   - 后端服务：不使用云服务
4. 点击「确定」创建

### Step 4：用 Cursor 打开项目

1. 打开 Cursor
2. File → Open Folder
3. 选择刚才创建的小程序项目文件夹
4. 现在你可以在 Cursor 里看到小程序的文件了

### 小程序项目结构（简单了解）

```
我的小程序/
├── pages/           # 页面文件夹
│   └── index/       # 首页
│       ├── index.js     # 逻辑
│       ├── index.wxml   # 结构（类似 HTML）
│       ├── index.wxss   # 样式（类似 CSS）
│       └── index.json   # 配置
├── app.js           # 小程序逻辑
```

## 3.5 发布上线

### 教学目标
- 了解小程序审核要求
- 完成发布流程
- 处理审核问题

### 发布前自查清单

在提交审核前，检查这些：

- [ ] 所有功能都能正常使用
- [ ] 在真机上测试过（不只是模拟器）
- [ ] 没有空白页面或明显 bug
- [ ] 没有违规内容（政治、色情、赌博等）
- [ ] 小程序有明确的功能，不是空壳
- [ ] 如果收集用户信息，有隐私说明

### 提交审核流程

**Step 1：上传代码**
1. 在微信开发者工具中
2. 点击右上角「上传」
3. 填写版本号（如 1.0.0）和备注
4. 点击确定

**Step 2：提交审核**
1. 登录 https://mp.weixin.qq.com
2. 进入「管理」→「版本管理」
3. 在「开发版本」中找到刚上传的版本
4. 点击「提交审核」
5. 填写小程序类目、截图等信息
6. 提交

**Step 3：等待审核**
- 通常 1-7 天
- 会收到审核结果通知
- 通过后点击「发布」即可上线

### 审核常见问题及解决

**问题：功能过于简单**
```
你的小程序功能单一，无法为用户提供有效服务。

解决：增加更多功能，让小程序更完整
```

**问题：缺少类目资质**
```
你选择的类目需要提供相关资质。

解决：更换为不需要资质的类目（工具类通常不需要）
```

**问题：内容违规**
```
存在违规内容。

解决：检查文字、图片是否有敏感内容，删除后重新提交
```

**问题：诱导分享**
```
存在诱导分享行为。

解决：不要强制用户分享才能使用功能
```

### 发布成功！

审核通过后：
1. 在版本管理中点击「发布」
2. 你的小程序正式上线了！
3. 可以在微信中搜索小程序名称找到它
4. 分享给朋友们吧！

## 4.1 H5 网页

### 什么是 H5

H5 就是网页，可以在任何浏览器打开，也可以分享到微信里。

**H5 vs 小程序**

| 对比项 | H5 网页 | 微信小程序 |
|--------|---------|------------|
| 打开方式 | 浏览器/微信链接 | 微信内 |
| 审核 | 不需要审核 | 需要审核 |
| 功能限制 | 较少限制 | 有平台限制 |
| 分享 | 分享链接 | 分享卡片 |

### 快速生成一个 H5 页面

在 Cursor 中：

```
帮我创建一个个人介绍 H5 页面：
- 单页面设计
- 包含：头像、姓名、简介、联系方式
- 风格现代，适合手机浏览
- 创建 index.html 文件，CSS 写在里面
```

### 免费发布 H5

**方案 1：GitHub Pages（推荐）**

1. 注册 GitHub 账号
2. 创建一个新仓库（Repository）
3. 上传你的 HTML 文件
4. 在仓库 Settings → Pages 中开启
5. 获得免费网址：`你的用户名.github.io/仓库名`

> 让 AI 教你：「请一步步教我把这个 HTML 发布到 GitHub Pages」

**方案 2：Vercel**

1. 注册 Vercel 账号
2. 导入 GitHub 仓库或直接上传
3. 自动生成网址
